
以下に従って Pull Request の指摘コメントに対応してほしい。

- Resolve されていないコメントを検索して、Resolve されていない **コメントごとに** 以下を繰り返す。
  - 指摘の方が間違っている時は間違っている旨を返信コメントして。
  - 指摘が正しい場合は
    - コメントごとに、該当コードを修正してコミットする
    - 対応した指摘のコメントには **それぞれの指摘コメントごとに** 対応済みの旨と修正コミットを返信コメントして。
      - 返信コメント方法 参考: https://docs.github.com/en/rest/pulls/comments?utm_source=chatgpt.com&apiVersion=2022-11-28#create-a-reply-for-a-review-comment
- コメント全て見終わったら push する。

## 例

### before

- hoge.kt#L10 このコードは不要です。削除すべきです。
- hoge.kt#L10 この記法を使うとよりクリーンに書けます。
  ```diff
  -"""some text"""
  +"some text"
  ```
- ファイル構造をよりシンプルにすべきです。

### after

- hoge.kt#L10 このコードは不要です。削除すべきです。
  - (返信) commit: [xxxxx](コミットへのリンク) で対応しました。
- hoge.kt#L10 この記法を使うとよりクリーンに書けます。
  ```diff
  -"""some text"""
  +"some text"
  ```
  - (返信) commit: [xxxxx](コミットへのリンク) で対応しました。
- ファイル構造をよりシンプルにすべきです。
  - (返信) これは~~のような意図がありこのようになっているため、特には変更はしませんでした。
