# Use cases

Practical recipes showing where cream.kt pays off in real projects. Each article starts from a plain hand-written implementation, shows how it degrades as requirements grow, and then rewrites it with cream.kt annotations so that only the meaningful diff remains. Every article is available in both English and Japanese.

## Overviews

- **[Managing UI state with sealed classes and cream.kt](./ui-state-management-by-sealed-class/README.md)** — Condensed overview of the five-part UiState series: `@CopyToChildren` for transitions between states and `@SealedCopy` for subtype-preserving updates. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/README.ja.md))*
- **[Simplifying model mapping across layers with cream.kt](./model-mapping.md)** — Hands the trivial data-layer → domain-layer copying to `@CopyMapping` so only the meaningful conversions (value classes, type changes) remain in your mapping functions. *([🇯🇵 日本語](./model-mapping.ja.md))*

## UI state management series

In-depth walkthroughs of the scenarios summarized in the overview above.

1. **[Part 1: Maintaining shared properties across Loading / Success / Error](./ui-state-management-by-sealed-class/01.md)** — `@CopyTo` on the transition-source state class removes the hand-copied shared properties from every state transition. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/01.ja.md))*
2. **[Part 2: Data-preserving transitions — refresh and optimistic updates](./ui-state-management-by-sealed-class/02.md)** — Replaces the ever-multiplying `when` for updating shared properties with the parent-typed `copy()` generated by `@SealedCopy`. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/02.ja.md))*
3. **[Part 3: Covering a nested sealed state machine with one annotation](./ui-state-management-by-sealed-class/03.md)** — One `@CopyToChildren` on the root generates copy functions for every transitive child class of a nested checkout-flow hierarchy. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/03.ja.md))*
4. **[Part 4: Writing MVI reducers declaratively](./ui-state-management-by-sealed-class/04.md)** — Combines `@CopyToChildren` and `@SealedCopy` so the reducer declares nothing but the per-event diffs. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/04.ja.md))*
5. **[Part 5: Using cream.kt with the Koma state-management library](./ui-state-management-by-sealed-class/05.md)** — Generated copy functions slot straight into Koma's state-machine DSL, so each `nextState` states only what actually changed. *([🇯🇵 日本語](./ui-state-management-by-sealed-class/05.ja.md))*
